# Data Dictionary

This repository contains CSV exports and normalized response datasets for Study 1: Top-Level Domain Signals, Signal Gap, and AI Source Triage.

## Core identifiers

|Field|Meaning|
|-|-|
|`study\_id`|Study identifier used in the source files.|
|`trial\_id`|Unique trial identifier where available.|
|`run\_id`|Replication/run identifier.|
|`model\_name` / `model`|AI model used for the response.|
|`timestamp`|Collection timestamp as recorded in the original data.|
|`domain`|Full domain stimulus.|
|`stem`|Semantically opaque domain stem.|
|`tld`|Top-level domain condition.|
|`prompt\_type` / `prompt\_version` / `prompt\_id`|Prompt condition identifiers.|

## Key coded dimensions

|Field|Meaning|
|-|-|
|`classification`|Sector classification output such as `medical`, `unclear`, or `other`.|
|`handling\_path` / `selected\_path`|Assigned triage path.|
|`risk\_weighting`|Low, medium, or high risk coding.|
|`justification\_type` / `explanation\_type`|Reasoning pattern or explanation category.|
|`difference\_flag`|Prompt 2 indicator for whether the model reported a handling difference.|
|`primary\_driver`|Main stated basis for the model decision.|
|`structural\_signal\_acknowledged`|Indicator for whether the model acknowledged the namespace-level signal.|
|`caution\_count`|Count of explicit hedging expressions under the study's coding rule.|
|`raw\_response` / `raw\_response\_text`|Model response text reconstructed to one row per trial.|

## Notes

* CSV files are UTF-8 encoded.
* Original Excel workbook tabs were exported to CSV, and formulas were resolved to stored values where applicable.
* Raw response files were reconstructed from line-oriented source sheets so that each trial appears as one row where possible.
* The `manifest.csv` file lists every file in this package and its purpose.

This repository contains CSV/xlxs exports, coded datasets, metadata files, and normalized response datasets for **Study 1: Signal Gap and Early-Stage Ambiguity Reduction in AI-Mediated Retrieval**.

The files document model behavior under controlled zero-content conditions designed to isolate the effects of namespace-level structural signals during early-stage source discovery and triage.

## Prompt-Specific Variables

Some datasets contain variables specific to individual prompt conditions.

### Prompt 1: Full Triage Assessment

Variables may include:

- subject-area classification
- risk weighting
- verification priority
- heightened scrutiny recommendation
- procedural handling path
- confidence estimates

### Prompt 1A: Retrieval Prioritization Under Uncertainty

Variables may include:

- retrieval priorities
- retrieval sequencing
- verification planning
- uncertainty-management strategy
- retrieval rationale

### Prompt 2: Comparative Procedural Handling

Variables may include:

- namespace comparison indicators
- reported handling differences
- stated procedural drivers

### Prompt 2B: Constrained Interpretation

Variables may include:

- structural-signal recognition
- constrained procedural reasoning
- routing and verification recommendations

### Prompt 3: Forced-Choice Procedural Routing

Variables may include:

- selected routing path
- forced-choice classification outcome
- procedural justification

## Signal Gap

The study defines **Signal Gap** as:

> The structural absence of sufficiently interpretable subject-area signals at the namespace layer during early-stage AI-mediated retrieval.

Signal Gap conditions were operationalized through semantically opaque fictitious domain labels paired with general-purpose namespace conditions.

## Replication Guidance

The datasets document model behavior observed during the reported testing period. Because frontier AI systems evolve over time, future outputs may differ due to model updates, interface changes, retrieval systems, alignment modifications, or other operational factors.

The study was designed to isolate namespace-level structural signals under zero-content conditions. Accordingly, the datasets reflect early-stage ambiguity reduction, source triage, retrieval planning, and procedural differentiation rather than end-to-end retrieval performance or downstream outcome quality.
